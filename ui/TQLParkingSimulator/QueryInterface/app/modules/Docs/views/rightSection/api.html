Get the list of all the organizations who manage Parking Lot

The purpose of this API is to get the list of all the Organization that are responsible for managing their respective Parking Lot. The result of this API can be used : - To retrieve the current parking lot snapshot. The “id” attribute value is used to formulate the download file URL of current parking lot snapshot.

Example:
http://tql.atomiton.com:8080/fid-ParkingResoures/res/Atom-Org-1/splatest.json
[Atom-Org-1 is the Organization “id” Value.]

API:
{{allOrganization}}
HTTP Method:
GET
Input JSON Format:
None
Output JSON Format:

Current Snapshot of the Parking Lot

The URL to retrieve initial/base version of the parking lot is:

http://tql.atomition.com:8080/fid-ParkingResources/res/<Org-Id>/splatest.json

Change the value of Org-Id with the value received as part of “get” Organization API call.

http://tql.atomiton.com:8080/fid-ParkingResources/res/Atom-Org-1/splatest.json

The purpose of this API is to get the data model associated with Smart Parking Lot. Knowing the model is useful in order to perform update / read operations on the individual.

HTTP Method:

None

Read All Individual Objects from a specific entity

The purpose of this API is to read all the objects that are part of a given entity. Example of Entitiy/Models are (from Get Smart Parking Data Model API): ParkingFloor, ParkingSpot, Camera, AreaLights, etc.

API:
{{allIndividualObjects}}
HTTP Method:
Post
Input JSON Format:
None

Read individual Objects from a specific entity and apply filters to select specific object

The purpose of this API is to read a specific object from the data model. For example, to Read the AreaLight with parkingSpotId = “Atom-Org-1.F10.S10” you will apply the filter to the Query string parameter – “FilterName”.

Format of FilterName itself is a JSON object as follows:
{“eq”: {“target”: “parkingFloorId”, “value”: “Atom-Org-1.F10.S10”}}

API:
{{allSpecificEntity}}
HTTP Method:
Post
Input JSON Format:
None

General Purpose Update on the specific data model

The purpose of this API is to perform an “update” on a specific data model. The update operation results in broadcast of an Event for the IoT Visualizer to make the User interface change.

Performing an update is a two step process:

1) Read the appropriate object using the read with FilterName.
2) Extract the sid (Unique internal Id of the object) and version of the attribute which is going to be part of the update operation.
3) Create a new filter that includes the version of the attribute and issue a update command.
