TQL Overview

Atomiton TQL System is a supervisory execution system for the Internet of Things (IoT). It enables people and applications to organize, access, and interact with large numbers of distributed but interconnected “things”, where “things” are defined as sensors, devices, machines, people, as well as passive resources instrumented with sensors

TQL System has three components: TQL Engine, IoT Simulator and TQL Studio.

TQL Engine is an event-driven, language-centric engine that allows the modeling/abstraction of things, the querying of things through such models, and interaction with things.

IoT Simulator is a cloud-based service that simulates how physical things interact with applications and/or interact with each other. The IoT Simulator simulates systems of things at scale from small to large, in order to help conceptualize, design, and test IoT applications.

TQLStudio brings the TQL Engine functions to the developer. It is a web application that allows TQL System users (developers, analyst, business owners) to model, query, and manage things in the context of building and testing IoT applications.

Using TQL to build IoT

1.Installation

You will start by installing JDK 1.7+ and downloads TQL System from the given site.

2.Author Thing Models Using TDL

Thing models are the logical representations of the “Things” in the IoT system. Thing models are entities with attributes, properties and operations. There are two types of thing models: data models and physical models.

Physical models have direct interactions with external physical entities, via sensing and/or actuating relationships. Data models do not have directly interaction with physical entities outside the software.

3.Develop Model Facets

Model facets are implementations that map between “Things” in the physical world and models in TQL. Physical models (as explained in section 2) need model facets to update states and/or actuate actions in the external world. Data models on the other hand, do not need model facets

4.Compile & Deploy Package

A Package contains the thing models and model facets required for a particular application. A package is a unit of deployment. Packages can be incrementally deployed.
